AI SKILLS

Claude Skills

Nine skills ship with NiCE Designer. Each one loads specialized context into Claude for a specific workflow - from triaging accessibility issues to converting a full Figma frame to Lyra tokens.

What is a skill?

Skills make Claude an expert tool operator

NiCE Designer ships 9 Claude skills. A skill is a context file (SKILL.md) that tells Claude exactly how to use the MCP tools for a specific job. You load a skill by saying "load the [skill-name] skill" in Claude chat, then Claude uses the right MCP tools in the right order.

Each skill pre-loads Claude with the correct tool sequence, common pitfalls, expected outputs, and quality checks for a particular workflow. Without a skill, Claude knows the tools exist but has to figure out the right approach on its own. With a skill loaded, it already has a playbook.

Skills are stored in the skills/ directory of the plugin. Each skill has a SKILL.md file. The same 9 skills are bundled into the packaged Chrome build at builds/chrome/skills/.

⚠️
Always call list_targets first to check which clients are connected (Figma, Chrome, or both). Some skills require both clients to be connected at the same time. Calling the wrong target silently does nothing.

MCP server

35 tools, 6 groups

NiCE Designer exposes 35 MCP tools via the local WebSocket server at 127.0.0.1:7331. All tools accept target: "figma" | "chrome" | "auto". Audit tools are read-only and safe to run any time. Conversion tools are undoable via undo_last_conversion.

Audit Read-only — safe to run any time, on either client
run_full_auditFigma + Chrome

Runs all audit checks at once and returns a combined compliance score with a full issue list across colors, text styles, spacing, radius, components, and accessibility.

Returns: overall score, per-category scores, issue count by severity

audit_colorsFigma + Chrome

Checks every color value in the selection against Lyra and SOL token palettes. Flags hardcoded hex and rgba values, and suggests the nearest token for each.

Returns: list of non-token colors with suggested replacements

audit_text_stylesFigma + Chrome

Identifies text nodes that don't use a named text style, and verifies that existing styles map to Lyra typography tokens rather than custom values.

Returns: unlinked text nodes, mismatched styles, token suggestions

audit_spacingFigma + Chrome

Audits padding, gap, and margin values against the 4pt spacing scale. Flags any values that don't land on a defined token and shows the nearest valid step.

Returns: off-grid spacing values with nearest token options

audit_radiusFigma + Chrome

Checks border-radius values across the selection against the Lyra radius token set. Flags any custom radius and suggests the closest token (none / sm / md / lg / full).

Returns: non-token radius values with suggested replacements

audit_componentsFigma only

Finds component instances and variants that don't match the linked Lyra component library. Shows version drift and detached overrides that break component fidelity.

Returns: mismatched components, detached instances, version diff

audit_accessibilityFigma + Chrome

Runs WCAG 2.1 AA checks: color contrast ratios, touch target minimums (44x44px), missing ARIA labels, focus order, and alt text coverage.

Returns: a11y issues grouped by type with WCAG criterion references

lyra_compliance_reportFigma + Chrome

Combines all audit results into the single Lyra compliance percentage score shown on the dashboard, with a per-category breakdown and a prioritized list of top fixes.

Returns: overall % score, category breakdown, priority fix list

Conversion Undoable — use undo_last_conversion to revert
convert_colorsFigma + Chrome

Replaces hardcoded color values with the nearest Lyra token. Dry-run by default — pass dryRun: false to apply.

Returns: preview of changes, count converted, unchanged values

convert_text_stylesFigma + Chrome

Applies the matching Lyra text style to every unlinked or mismatched text node. Matches on font size + weight + line-height to find the best token.

Returns: nodes updated, styles applied, nodes skipped (no match)

fix_spacingFigma + Chrome

Snaps padding and gap values to the nearest step on the 4pt spacing scale. Accepts a rounding mode: nearest, floor, or ceil.

Returns: values changed, new token value for each, unchanged count

fix_radiusFigma + Chrome

Replaces custom border-radius values with the nearest Lyra radius token. Preserves zero-radius (sharp) corners intentionally.

Returns: radius values updated, token applied to each, skipped count

convert_componentFigma only

Swaps a selected component instance to its Lyra library equivalent, preserving text and image overrides where the API allows.

Returns: old component name, new component name, overrides preserved / lost

fix_accessibilityFigma + Chrome

Auto-fixes common a11y issues: adds missing ARIA labels to icon-only buttons, adjusts text color for failing contrast where possible, and sets focus ring visibility.

Returns: fixes applied, issues that require manual intervention

convert_modeFigma + Chrome

Switches a frame or page between Lyra light and dark mode. In Figma, sets the variable mode. In Chrome, injects the appropriate token override stylesheet.

Returns: previous mode, new mode, variables or classes updated

convert_page_to_lyraChrome only

Applies Lyra CSS token overrides to the live web page. Non-destructive — injects a stylesheet that can be toggled off without touching the source.

Returns: properties overridden, before/after snapshot

migrate_to_lyraFigma only

Full SOL-to-Lyra migration on a selected frame: converts colors, text styles, spacing, radius, and component instances in a single pass. Large frames may take 10-30 seconds.

Returns: per-category change counts, items that couldn't be auto-migrated

generate_multi_systemFigma only

Duplicates the selected frame and generates two side-by-side versions: one styled in SOL and one in Lyra. Useful for compliance reviews or stakeholder comparisons.

Returns: node IDs of both generated frames, differences summary

undo_last_conversionFigma + Chrome

Reverts the most recent conversion operation. Only one undo level is stored — run immediately after a conversion if the result isn't right.

Returns: operation reverted, properties restored

Dev Handoff Read-only — extract specs and token CSS for engineering
extract_dev_detailsFigma only

Returns a full developer spec for the selected Figma node: dimensions, color fills, applied text styles, auto-layout properties, component props, and spacing values — formatted for handoff.

Returns: structured spec with all properties, token names where matched

extract_all_cssFigma + Chrome

Generates production CSS for the selected node. Hardcoded values are replaced with Lyra CSS custom property references where a token match exists.

Returns: CSS block with token variables, unresolved properties flagged

extract_token_cssFigma + Chrome

Exports the full Lyra or SOL token set as CSS custom properties in a :root {} block, ready to drop into a stylesheet or design token pipeline.

Returns: complete CSS variables file for the specified design system

Introspection Status and metadata — always call list_targets first
list_targetsFigma + Chrome

Shows which clients are currently connected to the MCP server. Run this first to confirm both the Figma plugin and Chrome extension are live before using multi-client tools.

Returns: connected clients list with version and status

get_selection_infoFigma + Chrome

Returns metadata about the current Figma selection or inspected Chrome element: node type, name, bounding box, applied styles, and component membership.

Returns: node/element metadata, current style values, component info

check_librariesFigma only

Verifies that the correct Lyra and SOL Figma libraries are linked in the current file and checks for outdated library versions that could cause component mismatches.

Returns: linked libraries, version status, missing or outdated flags

export_design_tokensFigma + Chrome

Exports all design tokens from the current Figma file, or the bundled Lyra / SOL token set, as a JSON object following the W3C design token format.

Returns: token JSON with name, value, type, and group for each token

get_diagnosticsFigma + Chrome

Returns connection health, plugin and extension versions, the current token hash (to verify token freshness), and any runtime errors logged by either client.

Returns: connection status, version strings, token hash, error log

Chrome Polish Live page QA overlays and token injection
polish_pageChrome only

Applies Lyra token overrides to the live page: correct color variables, Be Vietnam Pro font, and spacing tokens injected as a non-destructive CSS layer. Toggle off to restore originals.

Returns: properties overridden, overlay toggle state

a11y_overlayChrome only

Draws visual indicators on top of the live page: contrast ratios on text nodes, touch target outlines, focus ring visibility, and ARIA role labels. Useful during QA walkthroughs.

Returns: overlay toggled on/off, element count annotated

Escape Hatch Direct message pass-through — for advanced use only
call_pluginFigma + Chrome

Sends any raw UIMessage directly to the plugin or extension, bypassing the higher-level tool wrappers. Use when a workflow requires a message type that no existing tool exposes.

Returns: raw response from the plugin as-is

Start the MCP server: cd packages/mcp-server && node src/index.js — override the port with NICE_MCP_WS_PORT (default: 7331).

Tips

Before you run a skill

A few things to know that will save you time and prevent silent failures.

🔌
Always call list_targets first. Check which clients are connected before running any skill. Targeting Figma when only Chrome is connected does nothing and returns no error.
👥
Two clients required for: artboard-to-page and figma-chrome-parity. Both the Figma plugin and Chrome extension must be active and connected before running these skills.
🌐
Chrome-only skill: dark-mode-parity only works against a live page. There is no Figma equivalent. Run it with the Chrome extension active.
🏭
Figma-only skill: figma-element-css requires a node selected inside Figma. It cannot be run against Chrome page elements.
✨
Always load lyra-visual-patterns before any Lyra conversion. This sets Claude's expectations for token naming and visual patterns. Skipping it leads to inconsistent migration results.
🔒
Audit tools are safe anytime. All audit and introspection tools are read-only - they never modify your design. Run them freely. Conversion tools modify the design and support undo via undo_last_conversion.
