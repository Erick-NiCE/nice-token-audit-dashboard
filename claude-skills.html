AI SKILLS

Claude Skills

Nine skills ship with NiCE Designer. Each one loads specialized context into Claude for a specific workflow - from triaging accessibility issues to converting a full Figma frame to Lyra tokens.

What is a skill?

Skills make Claude an expert tool operator

NiCE Designer ships 9 Claude skills. A skill is a context file (SKILL.md) that tells Claude exactly how to use the MCP tools for a specific job. You load a skill by saying "load the [skill-name] skill" in Claude chat, then Claude uses the right MCP tools in the right order.

Each skill pre-loads Claude with the correct tool sequence, common pitfalls, expected outputs, and quality checks for a particular workflow. Without a skill, Claude knows the tools exist but has to figure out the right approach on its own. With a skill loaded, it already has a playbook.

Skills are stored in the skills/ directory of the plugin. Each skill has a SKILL.md file. The same 9 skills are bundled into the packaged Chrome build at builds/chrome/skills/.

⚠️
Always call list_targets first to check which clients are connected (Figma, Chrome, or both). Some skills require both clients to be connected at the same time. Calling the wrong target silently does nothing.

a11y-triage
● Figma or Chrome
Say something like
"triage accessibility issues" "which a11y issues matter most" "where do I start with a11y"

Groups audit_accessibility results by severity x reach and produces a prioritized action list instead of a raw issue dump. Helps you focus on fixes with the highest user impact first.

Key MCP tools
audit_accessibility

artboard-to-page
● Both clients required
Say something like
"apply this Figma frame to the live page" "push artboard styles to Chrome"

Takes a selected Figma artboard and applies its styling as CSS overrides onto the live Chrome page. Requires both Figma and Chrome clients to be connected at the same time.

Key MCP tools

browser-element-specs
● Chrome only
Say something like
"get specs for this element" "design specs for the inspected element" "what are the CSS values for this"

Generates a full developer spec sheet for an inspected Chrome extension element - spacing, type, colors, tokens, and interaction states. Ideal for handing off browser-side measurements to engineering.

Key MCP tools
extract_dev_details

dark-mode-parity
● Chrome only
Say something like
"check dark mode" "does this work in dark mode" "dark mode parity"

Tests whether the live page has a working dark mode, identifies which elements do not adapt, and reports what token changes are needed to achieve proper dark mode parity.

figma-chrome-parity
● Both clients required
Say something like
"diff Figma against the live page" "compare the design to the implementation" "figma-chrome parity"

Visually and structurally diffs a selected Figma artboard against the live implementing page. Reports mismatches in color, type, spacing, and layout. Requires both clients connected.

get_selection_info extract_all_css audit_colors audit_text_styles

figma-element-css
● Figma only
Say something like
"get CSS for this node" "export CSS for the selected element" "what's the CSS for this"

Exports production-ready CSS for the selected Figma node with Lyra design tokens substituted in for raw values. Gives developers clean, token-referenced CSS they can drop directly into code.

Key MCP tools
extract_all_css extract_token_css

● Figma or Chrome
Load BEFORE Lyra conversion
"load lyra-visual-patterns skill" "convert to Lyra" "migrate to Lyra"

Sets Claude's expectations for what a correct Lyra conversion looks like - token naming conventions, visual patterns, and common pitfalls. Always load this skill before running migrate_to_lyra or convert_page_to_lyra.

context-setter (used before migration tools)

manual-fixes-checklist
● Figma or Chrome
Say something like
"what can't be auto-fixed" "what's left to do manually" "what do I still need to fix by hand"

After running auto-fix tools, generates a checklist of issues that require human judgment - things the automation cannot handle safely. Helps you know exactly what still needs attention.

Key MCP tools

● Figma or Chrome
Say something like

Scans the selection or page for recurring non-token values - custom hex colors, arbitrary spacing values - that appear enough times to be worth promoting to design tokens. Great for design system maintenance.

Tips

Before you run a skill

A few things to know that will save you time and prevent silent failures.

🔌
Always call list_targets first. Check which clients are connected before running any skill. Targeting Figma when only Chrome is connected does nothing and returns no error.
👥
Two clients required for: artboard-to-page and figma-chrome-parity. Both the Figma plugin and Chrome extension must be active and connected before running these skills.
🌐
Chrome-only skill: dark-mode-parity only works against a live page. There is no Figma equivalent. Run it with the Chrome extension active.
🏭
Figma-only skill: figma-element-css requires a node selected inside Figma. It cannot be run against Chrome page elements.
✨
Always load lyra-visual-patterns before any Lyra conversion. This sets Claude's expectations for token naming and visual patterns. Skipping it leads to inconsistent migration results.
🔒
Audit tools are safe anytime. All audit and introspection tools are read-only - they never modify your design. Run them freely. Conversion tools modify the design and support undo via undo_last_conversion.
