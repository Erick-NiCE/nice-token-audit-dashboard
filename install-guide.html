GETTING STARTED

Install Guide

Get NiCE Designer running in three steps: the Figma plugin, the Chrome extension, and the local MCP server.

1
🎨

Install the Figma Plugin

How to install

Open Figma, right-click the canvas (or open the main menu) and go to Plugins. Search for "NiCE Designer" and click Install.

Or open the plugin directly on the Figma Community page:

figma.com/community/plugin/1623361730572479462

How to use it

With a frame or layer selected, right-click and go to Plugins > NiCE Designer. Alternatively, open the Resources panel in Figma (Shift+I), switch to the Plugins tab, and search for NiCE Designer.

Notes

The plugin runs entirely inside Figma. No backend or server is required for standalone audit and convert workflows. Connect to the MCP server to enable Claude integration and cross-client features.

2
🌐

Install the Chrome Extension

Prerequisites

Chrome or Edge (Chromium-based) with Developer Mode enabled in chrome://extensions.

How to install
  1. Download the latest release from the resources hub
  2. Unzip the downloaded package to a location you'll keep it
  3. Go to chrome://extensions and enable Developer Mode (toggle in the top-right corner)
  4. Click "Load unpacked" and select the chrome/ folder from the unzipped package
  5. The NiCE Designer icon appears in your Chrome toolbar
How to use it

Navigate to any web page and click the NiCE Designer icon in the toolbar. The inspection panel opens alongside the page. For a dedicated second-monitor view, click the pop-out icon in the extension settings panel.

ℹ️

The extension persists per window. No reset is needed when switching tabs or navigating to a different page.

3
🤖

Start the MCP Server

What it does

The MCP server is a local WebSocket hub running on port 7331 that connects Claude to both the Figma plugin and Chrome extension. Without it, Claude skills and MCP tools won't work. The Figma plugin and Chrome extension still function fully as standalone tools without the server.

Prerequisites

Node.js 18 or higher installed on your machine.

How to start
  1. The MCP server is included in the same downloaded package as the Chrome extension
  2. Navigate to the packages/mcp-server/ folder
  3. On first run, install dependencies: npm install
  4. Start the server: node src/index.js
  5. The server starts on ws://127.0.0.1:7331
  6. Open Claude Desktop or any MCP-compatible client and connect to ws://127.0.0.1:7331
Custom port

Set the NICE_MCP_WS_PORT environment variable before starting to use a different port:

NICE_MCP_WS_PORT=8080 node src/index.js
💡

Run npm install first on a fresh install to pull in ws and other dependencies.

4
🔌

Connect Claude Desktop Optional

Add to Claude Desktop config

Add the MCP server entry to your Claude Desktop configuration file at claude_desktop_config.json:

// claude_desktop_config.json { "mcpServers": { "nice-designer": { "command": "node", "args": ["/path/to/nice-designer/packages/mcp-server/src/index.js"] } } }

Use an absolute path to the index.js file. Relative paths will not work.

After saving

Restart Claude Desktop. You should see "nice-designer" appear as a connected MCP server in the interface.

Best practice

Load a skill before using MCP tools for the best results. For example, tell Claude: "Load the lyra-visual-patterns skill" before starting a Lyra conversion workflow.

5
✅

Verify Everything Is Working

Quick checks
Open the Figma plugin on a frame - the audit panel should appear with color, type, and spacing scores
Open the Chrome extension on a web page - the toolbar should appear with inspection controls
Start the MCP server, open Claude Desktop, and check the MCP server indicator shows "nice-designer" as connected
Ask Claude: "Call list_targets" - it should respond listing which clients are currently connected (Figma, Chrome, or both)
Troubleshooting

Common Issues

🎨 Plugin won't open
Make sure you installed from the Figma Community (not a local unpacked version). Check that the plugin isn't in your organization's blocked plugin list - contact your Figma admin if unsure.
🌐 Chrome extension not loading
Verify that Developer Mode is enabled in chrome://extensions. If the extension still won't load, try removing it and re-loading the unpacked folder. Make sure you're selecting the chrome/ subfolder, not the root of the zip.
🔌 MCP server not connecting
Check that port 7331 is not already in use by another process. Use NICE_MCP_WS_PORT=7332 node src/index.js to run on a different port. Make sure the path in claude_desktop_config.json is an absolute path, not relative.
🤖 Claude can't see any targets
Both the Figma plugin and Chrome extension must be open and connected to the MCP server at the same time. Call list_targets first - it will tell you exactly which clients are currently registered. If the list is empty, check the connections in each client.
