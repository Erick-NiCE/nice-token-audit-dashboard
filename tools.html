The Suite

8 product tools

The high-level tools that make up NiCE Designer. Each one is available in the Figma plugin, Chrome extension, or both — and some unlock additional capabilities when connected via Superpowers.

Audit Read-only — Figma + Chrome
🔍
Audit Tool
Runs a full Lyra compliance check on any selected Figma frame or live Chrome page. Scores colors, text styles, spacing, radius, components, and accessibility in one pass and returns a percentage score.
Figma Chrome Read-only Dashboard score
Convert Undoable — Figma + Chrome
⚙️
Convert Tool
Replaces non-token values with the nearest Lyra design token. Works on colors, text styles, spacing, and radius — surgically or across an entire frame. All conversions are undoable in one step.
Figma Chrome Undoable Dry-run mode
Handoff Read-only — Figma primary
📋
Handoff Tool
Generates a full developer spec from the selected Figma node: dimensions, fills, typography, spacing, component props — all with Lyra token names substituted in. Also exports production-ready CSS.
Figma CSS export Token CSS Dev spec
Parity Requires Superpowers — both clients
⚖️
Parity Tool
Diffs a Figma artboard against the live implementing page in Chrome. Flags visual gaps: color drift, spacing mismatches, font differences, and missing components. Requires both clients connected.
Figma Chrome ⚡ Superpowers
QA Chrome only

Live page quality checks in Chrome. Toggle dark mode, run colorblind simulation, highlight touch targets, show a11y overlays with contrast ratios and ARIA labels, and polish the page with Lyra tokens.
Chrome only Dark mode A11y overlay Colorblind sim

MCP Server

35 MCP tools

Called directly by Claude when Superpowers is connected. All tools accept target: "figma" | "chrome" | "auto". Start the server with cd packages/mcp-server && node src/index.js (default port 7331).

Audit Read-only — safe any time
run_full_auditFigma + Chrome

Runs all audit checks at once and returns a combined compliance score with a full issue list across colors, text, spacing, radius, components, and accessibility.

Returns: overall score, per-category scores, issue count by severity

audit_colorsFigma + Chrome

Checks every color value against Lyra and SOL token palettes. Flags hardcoded hex and rgba values and suggests the nearest token for each.

Returns: non-token colors with suggested replacements

audit_text_stylesFigma + Chrome

Identifies text nodes that don't use a named text style, and verifies that existing styles map to Lyra typography tokens.

Returns: unlinked text nodes, mismatched styles, token suggestions

audit_spacingFigma + Chrome

Audits padding, gap, and margin values against the 4pt spacing scale. Flags values that don't land on a defined token.

Returns: off-grid spacing values with nearest token options

audit_radiusFigma + Chrome

Checks border-radius values against the Lyra radius token set. Flags custom values and suggests the closest token (none / sm / md / lg / full).

Returns: non-token radius values with suggested replacements

audit_componentsFigma only

Finds component instances and variants that don't match the linked Lyra library. Shows version drift and detached overrides.

Returns: mismatched components, detached instances, version diff

audit_accessibilityFigma + Chrome

Runs WCAG 2.1 AA checks: color contrast ratios, touch target minimums (44×44px), missing ARIA labels, focus order, and alt text coverage.

Returns: a11y issues grouped by type with WCAG criterion references

lyra_compliance_reportFigma + Chrome

Combines all audit results into the single Lyra compliance percentage shown on the dashboard, with a per-category breakdown and prioritized top fixes.

Returns: overall % score, category breakdown, priority fix list

Conversion Undoable — use undo_last_conversion to revert
convert_colorsFigma + Chrome

Replaces hardcoded color values with the nearest Lyra token. Dry-run by default — pass dryRun: false to apply.

Returns: preview of changes, count converted

convert_text_stylesFigma + Chrome

Applies the matching Lyra text style to every unlinked or mismatched text node, matching on font size + weight + line-height.

Returns: nodes updated, styles applied, skipped count

fix_spacingFigma + Chrome

Snaps padding and gap values to the nearest step on the 4pt spacing scale. Accepts rounding mode: nearest, floor, or ceil.

Returns: values changed, new token for each

fix_radiusFigma + Chrome

Replaces custom border-radius values with the nearest Lyra token. Preserves intentional zero-radius (sharp) corners.

Returns: values updated, token applied to each

convert_componentFigma only

Swaps a selected component instance to its Lyra library equivalent, preserving text and image overrides where the API allows.

Returns: old/new component names, overrides preserved or lost

fix_accessibilityFigma + Chrome

Auto-fixes common a11y issues: adds missing ARIA labels to icon-only buttons, adjusts text color for failing contrast where possible, and sets focus ring visibility.

Returns: fixes applied, issues requiring manual intervention

convert_modeFigma + Chrome

Switches a frame or page between Lyra light and dark mode. In Figma, sets the variable mode. In Chrome, injects the token override stylesheet.

Returns: previous mode, new mode, variables updated

convert_page_to_lyraChrome only

Applies Lyra CSS token overrides to the live web page via a non-destructive injected stylesheet that can be toggled off without touching source.

Returns: properties overridden, before/after snapshot

migrate_to_lyraFigma only

Full SOL-to-Lyra migration on a selected frame: converts colors, text styles, spacing, radius, and components in a single pass. Large frames may take 10-30 seconds.

Returns: per-category change counts, items needing manual review

generate_multi_systemFigma only

Duplicates a frame and generates two side-by-side versions: one in SOL and one in Lyra. Useful for stakeholder comparisons or compliance reviews.

Returns: node IDs of both frames, differences summary

undo_last_conversionFigma + Chrome

Reverts the most recent conversion operation. One undo level is stored — run immediately after a conversion if the result isn't right.

Returns: operation reverted, properties restored

Dev Handoff Read-only — extract specs and CSS
extract_dev_detailsFigma only

Returns a full developer spec for the selected node: dimensions, color fills, text styles, auto-layout properties, component props, and spacing — formatted for handoff.

Returns: structured spec with all properties, token names where matched

extract_all_cssFigma + Chrome

Generates production CSS for the selected node. Hardcoded values are replaced with Lyra CSS custom property references where a token match exists.

Returns: CSS block with token variables, unresolved properties flagged

extract_token_cssFigma + Chrome

Exports the full Lyra or SOL token set as CSS custom properties in a :root {} block, ready to drop into a stylesheet or design token pipeline.

Returns: complete CSS variables file for the specified system

Introspection Status and metadata — call list_targets first
list_targetsFigma + Chrome

Shows which clients are currently connected to the MCP server. Always run this first to confirm both clients are live before multi-client tools.

Returns: connected clients list with version and status

get_selection_infoFigma + Chrome

Returns metadata about the current Figma selection or Chrome element: node type, name, bounding box, applied styles, and component membership.

Returns: node metadata, current style values, component info

check_librariesFigma only

Verifies that the correct Lyra and SOL Figma libraries are linked in the current file and checks for outdated library versions.

Returns: linked libraries, version status, missing or outdated flags

export_design_tokensFigma + Chrome

Exports all design tokens from the current Figma file or the bundled Lyra / SOL token set as JSON following the W3C design token format.

Returns: token JSON with name, value, type, and group

get_diagnosticsFigma + Chrome

Returns connection health, plugin and extension versions, the current token hash (to verify freshness), and any runtime errors logged by either client.

Returns: connection status, version strings, token hash, error log

Chrome Polish Live page QA overlays and token injection
polish_pageChrome only

Applies Lyra token overrides to the live page: correct color variables, Be Vietnam Pro font, and spacing tokens as a non-destructive CSS layer. Toggle off to restore originals.

Returns: properties overridden, overlay toggle state

a11y_overlayChrome only

Draws visual indicators on the live page: contrast ratios on text nodes, touch target outlines, focus ring visibility, and ARIA role labels. Useful during QA walkthroughs.

Returns: overlay toggled on/off, element count annotated

Escape Hatch Advanced — direct message pass-through
call_pluginFigma + Chrome

Sends any raw UIMessage directly to the plugin or extension, bypassing higher-level wrappers. Use when no existing tool covers the needed message type.

Returns: raw response from the plugin as-is

Claude Skills

9 skills

Skills are context files (SKILL.md) that give Claude a playbook for a specific workflow. Load a skill by saying "load the [skill-name] skill" in Claude chat — then Claude uses the right MCP tools in the right sequence. All skills require Superpowers.

a11y-triage
Figma + Chrome

Groups accessibility issues from audit_accessibility by severity times reach, then produces a top-N action list. Instead of a raw issue dump, you get a prioritized list of what to fix first for the most user impact.

Trigger: "triage a11y issues", "which a11y issues matter most", "where do I start with accessibility"

artboard-to-page
Figma + Chrome

Takes a selected Figma artboard and pushes its styling onto the live Chrome page as CSS overrides. Both clients must be connected. Use this to preview a design direction on real content without rebuilding anything.

Trigger: "apply this artboard to the live page", "push Figma styles to Chrome"

browser-element-specs
Chrome only

Inspects a Chrome element and returns a Figma-ready design spec: dimensions, colors mapped to Lyra tokens, typography, spacing, and radius. Useful when reverse-engineering a page into a design system component.

Trigger: "give me the specs for this element", "what are the design values on this component"

dark-mode-parity
Chrome only

Toggles the live page between light and dark mode and audits for visual issues: color contrast failures, hardcoded light-mode values that don't adapt, and missing dark-mode token bindings.

Trigger: "check dark mode", "does this page work in dark mode", "dark mode parity check"

figma-chrome-parity
Figma + Chrome

Diffs the selected Figma artboard against the live Chrome page. Flags color drift, spacing mismatches, font differences, and missing or substituted components. Requires both clients connected.

Trigger: "diff Figma against the live page", "how close is the implementation to the design"

figma-element-css
Figma only

Exports production CSS for the selected Figma node with Lyra token variable names substituted in. Goes further than Figma's native inspect panel — includes auto-layout as flexbox and maps every value to its token.

Trigger: "get CSS for this node", "export Figma CSS with tokens"

lyra-visual-patterns
Figma + Chrome

Loads Lyra's visual pattern expectations into Claude's context: how tokens should be applied, common layout patterns, component usage rules, and what a compliant frame looks like. Load this before any Lyra conversion for better results.

Trigger: load automatically before convert or migrate operations

manual-fixes-checklist
Figma + Chrome

After all auto-fixers have run, identifies what still needs human attention: complex component mismatches, custom illustrations, nuanced a11y decisions, and anything the automation can't safely change. Returns a prioritized manual checklist.

Trigger: "what's left to fix manually", "what can't be auto-fixed", "show me the manual checklist"

token-candidates
Figma + Chrome

Finds custom values (colors, spacing, radius, type) that appear frequently enough across the file or page to be worth promoting to a Lyra token. Returns a shortlist with usage counts and suggested token names.

Trigger: "what custom values should be tokens", "find token candidates", "which values repeat most"
