Use Cases

Everything NiCE Designer can do, organized by job to be done. From auditing a Figma frame to pushing a design live, each scenario tells you which tools to use and in what order.

Read-only

Auditing — Is this compliant?

🔍

Full compliance audit on a Figma frame

✦ Figma

When you want to know if a frame meets SOL or Lyra standards before handoff.

  1. 1Select the target frame in Figma.
  2. 2Open the plugin and click the Audit tab, or ask Claude to run a full audit.
  3. 3Review separate scores for Colors, Text, Spacing, Radius, Components, and two a11y scores.
run_full_audit
🌐

Full compliance audit on a live web page

○ Chrome

When you want to know if a shipped page matches the design system.

  1. 1Open the Chrome extension on the target page.
  2. 2Click the Audit tab, or ask Claude to run a full audit against the Chrome target.
  3. 3Review the real-time scores and issue list for the rendered page.
run_full_audit target: chrome
🎯

Single-dimension audit

▲ Figma or Chrome

When you only care about one aspect, for example after fixing all colors and wanting to verify only colors.

  1. 1In the plugin, switch to the per-tab audit: Colors, Text, Spacing, Radius, or Components.
  2. 2Or ask Claude to run a specific audit tool for just that dimension.
  3. 3Review focused results without noise from other categories.
audit_colors audit_text_styles audit_spacing audit_radius audit_components
⚠

Accessibility triage

▲ Figma or Chrome

When you have a long a11y issue list and need to know where to start.

  1. 1Load the a11y-triage skill in Claude.
  2. 2Ask Claude to audit and triage accessibility issues.
  3. 3Review the output: issues grouped by severity x reach with a prioritized action list.
a11y-triage audit_accessibility
2
Write operations

Converting — Make it compliant

🏭

Convert a whole Figma frame to Lyra tokens

✦ Figma

When you have a frame built with raw values or SOL tokens and need it in Lyra.

  1. 1Load the lyra-visual-patterns skill first to set expectations.
  2. 2Ask Claude to migrate, or use the Convert tab in the plugin.
  3. 3Choose a mode: tokens-only (safe, swaps tokens and preserves layout) or full (also normalizes layout, components, and a11y).
lyra-visual-patterns migrate_to_lyra
🌏

Convert a live web page to Lyra tokens

○ Chrome

When you want to re-theme a shipped page to use Lyra CSS variables.

  1. 1Open the Chrome extension on the target page.
  2. 2Ask Claude to convert the page to Lyra, or use the Convert tab in the extension.
  3. 3Review the token substitutions applied as CSS variable overrides.
convert_page_to_lyra
🔧

Surgical per-element fixes

▲ Figma or Chrome

When you want to fix individual issues rather than convert everything at once.

  1. 1In the plugin issue list, click Replace on specific rows, or ask Claude to fix a single property.
  2. 2Each fix is logged in the edit history and can be undone per step (since v10).
convert_colors fix_spacing fix_radius convert_text_styles
🔄

SOL to Lyra swap

✦ Figma

When a frame was built in SOL and needs a Lyra version, or vice versa.

  1. 1Ask Claude to generate both system variants side by side, or convert the existing frame.
  2. 2Review the generated variants and choose the one that matches your target system.
generate_multi_system convert_mode
↩

Undo a conversion

▲ Figma or Chrome

When a conversion didn't look right and you want to step back.

  1. 1Use the edit history panel in the plugin (undo/redo), or ask Claude to undo.
  2. 2Each step is individually revertible, so you can roll back just the last change.
undo_last_conversion

Export and specs

Design-to-code handoff

📄

Export a Figma node as production CSS

✦ Figma

When engineering needs the exact CSS for a component with design tokens substituted in for raw values.

  1. 1Select the node in Figma.
  2. 2Load the figma-element-css skill, then ask Claude to export CSS.
  3. 3Receive production-ready CSS with Lyra token variables in place of raw values.
figma-element-css extract_all_css extract_token_css
📊

Get a developer spec sheet for a browser element

○ Chrome

When engineering inspects a live element and needs all its design properties in one place.

  1. 1In Chrome, inspect the element via the extension.
  2. 2Load the browser-element-specs skill, then ask Claude for specs.
  3. 3Receive a spec sheet covering spacing, type, colors, tokens, and interaction states.
browser-element-specs extract_dev_details
👔

Export the full token set

▲ Figma or Chrome

When you need a token export for SOL or Lyra, for example to feed into a style dictionary pipeline.

  1. 1Ask Claude to export tokens, specifying the target system (SOL or Lyra).
  2. 2Receive a structured token export ready for downstream tooling.
export_design_tokens
4
Both clients required

Figma vs. live page parity

🔗 These workflows require both the Figma plugin and the Chrome extension to be connected to the MCP server simultaneously. Start the MCP server first (cd packages/mcp-server && node src/index.js), then open both clients.
🔎

Diff a Figma artboard against the live page

▲ Both (required)

When you suspect the implementation doesn't match the design and need a concrete diff.

  1. 1Open both the Figma plugin and the Chrome extension and connect both to MCP.
  2. 2Load the figma-chrome-parity skill.
  3. 3Ask Claude to compare. Receive a side-by-side mismatch report for colors, type, spacing, and layout.
figma-chrome-parity
🖌

Push Figma styling onto the live page

▲ Both (required)

When you want to preview what the page would look like if it matched the design exactly.

  1. 1Connect both clients to MCP and select the source frame in Figma.
  2. 2Load the artboard-to-page skill.
  3. 3Ask Claude to apply. CSS overrides matching the Figma frame are applied to the live page instantly.
artboard-to-page

QA and maintenance

Chrome page polish and QA

🌕

Dark mode parity check

○ Chrome only

When you need to verify the page works correctly in dark mode before shipping.

  1. 1Load the dark-mode-parity skill.
  2. 2Ask Claude to check dark mode on the current page.
  3. 3Receive a report of which elements don't adapt correctly and what token changes are needed.
dark-mode-parity
👀

Toggle visual overlays for QA

○ Chrome

When you want visual debugging during a QA pass without leaving the browser.

  1. 1Open the Chrome extension's QA panel, or ask Claude to toggle a specific overlay.
  2. 2Choose from: scrollbar overlay, hover overlay, tap-target highlighting, or colorblind simulation.
  3. 3Toggle off overlays when your QA pass is complete.
a11y_overlay polish_page
👑

Find recurring values worth tokenizing

▲ Figma or Chrome

When you're maintaining the design system and want to clean up custom values that should be tokens.

  1. 1Load the token-candidates skill.
  2. 2Ask Claude to find recurring non-token values.
  3. 3Receive values grouped by frequency with suggested token names for each.
token-candidates export_design_tokens
✅

After auto-fixing: what's left to do manually

▲ Figma or Chrome

When you've run all the fix tools and want to know what still requires human judgment.

  1. 1Load the manual-fixes-checklist skill after running your automated fixes.
  2. 2Ask Claude to generate the checklist.
  3. 3Receive a human-readable checklist of issues that automation can't safely resolve.
manual-fixes-checklist
